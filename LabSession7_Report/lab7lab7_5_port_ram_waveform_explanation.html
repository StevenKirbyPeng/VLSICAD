二、Port List of Draw Bounding Box Engine 說明

Port list 是本實驗最重要的介面規格。設計者可以自由實作 drawBbox 內部架構，例如加入灰階轉換、Gaussian filter、Otsu threshold、mask polarity correction、8-connected CCL、bounding box 與 overlay 等模組，但不能任意更改這些 I/O port 的名稱、方向與位元寬度。因為 testbench 會依照這些 port 連接 DUT，若 port 不一致，模擬會直接編譯失敗或連接錯誤。

SignalTypeBits功能說明

doneoutput1處理完成旗標。當 256×256 影像處理、bounding box 繪製與 AnsSRAM 寫入完成後拉高。

三、RAM Read Operation Waveform 詳細說明

第一張波形圖是 RAM 讀取操作範例。讀取時，CEN 必須為低電位，表示 SRAM 被啟用；WEN 維持高電位，表示目前是 read mode，不是 write mode。位址 A 在 clock 的 positive edge 被鎖存，資料 Q 會在後續時間輸出對應位址的記憶體內容。

CEN = 0 啟用 SRAM→ WEN = 1 選擇讀取→ posedge clk 鎖存 address→ Q 輸出 Mem[A]

讀取波形中的各訊號意義

  • clk：SRAM 以 clock 邊緣為基準進行同步操作。
  • CEN：低電位有效。當 CEN 拉低時，SRAM 才會接受 address 並進行讀取。
  • WEN：高電位代表 read mode。因為沒有寫入，所以 D 顯示為 Don’t Care。
  • A：讀取位址，例如圖中的 Addr R1、Addr R2。
  • Q：讀取資料輸出，例如 Mem[R1]、Mem[R2]。圖中標示資料大多在 negative edge 附近可觀察到穩定輸出。

在 Lab7 的設計中，read operation 會用於從 ImgROM 讀取原始影像、從 UrSRAM 讀回中間資料，或從 AnsSRAM 讀取先前寫入結果。若資料有一拍或多拍 latency，FSM 必須加入 read wait state，避免 address 剛送出就立刻使用尚未穩定的 Q。

四、RAM Write Operation Waveform 詳細說明

第二張波形圖是 RAM 寫入操作範例。寫入時，CEN 同樣必須為低電位以啟用 SRAM；WEN 則必須拉低，表示目前是 write mode。此時設計必須在 clock 邊緣前後提供穩定的 address A 與 write data D，SRAM 才能把資料正確寫進指定位址。

CEN = 0 啟用 SRAM→ WEN = 0 選擇寫入→ A 指定寫入位址→ D 提供寫入資料→ Mem[A] 更新

寫入波形中的各訊號意義

  • clk：寫入動作依照同步時脈進行。
  • CEN：低電位有效。若 CEN 沒有拉低，即使 WEN=0 也不應該完成有效寫入。
  • WEN：低電位代表 write mode，是判斷讀寫模式的關鍵訊號。
  • A：寫入位址，例如 Addr W1、Addr W2。
  • D：寫入資料，例如 Data W1、Data W2，必須在寫入時保持穩定。
  • Q：有些 SRAM 會在寫入後讀出新資料，因此圖中可看到 Mem[W1]、Mem[W2] 出現在 Q 端。

在 Draw Bounding Box Engine 中，write operation 最常出現在兩個地方：一是把中間處理結果寫入 UrSRAM，例如灰階、mask 或暫存影像；二是把最後畫好綠色 bounding box 的 overlay 結果寫入 AnsSRAM。為了讓輸出 bitmap 正確，address 與 data 必須完全對齊，否則會產生影像偏移、框線錯位或 pixel compare error。

五、Lab7 實作時的檢查重點

1、Port 不可改名
最外層 drawBbox port 必須與 testbench 規格完全一致。
2、Active Low 要特別注意
CEN 是低有效，WEN=0 是寫入，WEN=1 是讀取。
3、Address 與 Data 要對齊
讀取或寫入都要考慮 SRAM/ROM latency，避免 pixel 位移。
4、AnsSRAM 是最終答案
最後輸出的 32-bit pixel 必須寫到 Ans_D，並使用正確的 Ans_A。
5、UrSRAM 可作中間暫存
若設計需要多階段處理，可用 UrSRAM 儲存中間影像或 mask。
6、done 拉高時機
必須等所有 pixel 都寫入完成後再拉高，不能太早結束。

六、可放入報告的總結文字

本附件說明 Draw Bounding Box Engine 的外部介面與 RAM 存取時序。Port list 定義了 DUT 與 testbench 中 ImgROM、UrSRAM、AnsSRAM 的連接方式，其中 ImgROM 負責提供輸入影像，UrSRAM 可作為中間資料儲存空間，AnsSRAM 則儲存最後的 bounding box overlay 結果。RAM read waveform 顯示當 CEN=0 且 WEN=1 時，位址會在 clock 邊緣被鎖存並於輸出端 Q 取得資料；RAM write waveform 顯示當 CEN=0 且 WEN=0 時，輸入資料 D 會寫入位址 A 所指定的記憶體位置。此時序規則對 Lab7 非常重要，因為任何 address、data 或 control signal 的錯位都可能造成影像偏移、框線錯誤或無法通過 pixel-perfect comparison。
