一、附件文字內容說明

原文提到：the design inside the bicubic block can be completed by your free will, but do not modify the I/O ports of the bicubic block。在本 Lab 7 圖中，這句話的實際意思可以理解為：DUT 內部演算法與模組切分可以自由設計，但最外層模組名稱與訊號介面必須符合作業規格。圖中雖然文字寫到 bicubic block，但此頁主題是 Draw Bounding Box，因此應以 drawBbox 作為本 Lab 的 DUT 來理解。

重點：老師或 TA 的 tb.sv 會直接 instance 你的 drawBbox。如果你更改 port 名稱、方向或 bit 數，即使內部演算法正確，也可能無法編譯或無法通過模擬。

二、整體架構：Testbench 與 DUT

testbench

上方藍色大區塊代表測試環境。它負責產生 clock/reset/enable，提供輸入影像記憶體 ImgROM，並建立中間或結果記憶體 UrSRAM、AnsSRAM，最後檢查輸出的 bitmap 或 golden result。

drawBbox

下方橘色大區塊是學生要實作的主設計，也就是 DUT。所有影像處理、threshold、CCL、bounding box 計算與 overlay 寫回，都應在此模組或其子模組中完成。

三、三個記憶體區塊功能

區塊功能在 Lab 7 中的角色
ImgROM輸入影像記憶體，只讀。存放 256×256 RGB 原始影像。drawBbox 透過 Img_CEN、Img_A 讀取 pixel，從 Img_Q 取得 32-bit RGB data。
UrSRAM中間結果 SRAM，可讀可寫。可用來暫存灰階、Gaussian 結果、mask、label 或其他中間資料。是否大量使用取決於你的架構。
AnsSRAM最終答案 SRAM，可讀可寫。必須寫入最後 overlay 結果，也就是原圖加上綠色 bounding box 的輸出影像。

四、主要 I/O 訊號說明

訊號類別訊號說明
控制訊號clk, rst, enable, doneclk 控制所有同步電路；rst 重設狀態；enable 啟動處理；done 表示整張圖處理完成。
ImgROM 介面Img_CEN, Img_A, Img_QImg_CEN 為 active low read enable；Img_A 是讀取位址；Img_Q 是從 ROM 回傳的 32-bit pixel data。
UrSRAM 介面Ur_WEN, Ur_CEN, Ur_A, Ur_D, Ur_Q中間資料 SRAM。WEN=0 通常表示寫入，WEN=1 表示讀取；CEN 為 active low chip enable。
AnsSRAM 介面Ans_WEN, Ans_CEN, Ans_A, Ans_D, Ans_Q最終答案 SRAM。你需要把每個 pixel 的最終 32-bit 顏色寫到 Ans_D，位址由 Ans_A 控制。

五、drawBbox 內部可以自由設計的部分

圖中 drawBbox 內部放了一個虛線綠色方塊 ccl_engine，表示這只是其中一種可能的內部設計。你可以把 CCL 獨立成子模組，也可以把整個流程全部寫在 drawBbox.v 裡。只要外部介面不變，並且結果能通過 RTL simulation、gate-level simulation 與 pixel comparison，即符合此圖的精神。

可自由修改
  • FSM 狀態數量與狀態名稱
  • 是否使用子模組
  • 是否使用 UrSRAM 暫存中間結果
  • CCL label 資料結構
  • Bounding box 過濾與 overlay 實作方式
不可任意修改
  • drawBbox 的 port 名稱
  • port 的 input/output 方向
  • port 的 bit width
  • testbench 預期的 memory protocol
  • done 完成時機與輸出資料格式

六、建議的影像處理流程

  1. 讀取 ImgROM：依 row-major address 逐點讀取 256×256 pixel。
  2. RGB to Grayscale：將 RGB 轉為單一灰階強度，方便 threshold。
  3. Gaussian Filter：降低雜訊，讓 threshold 與 CCL 更穩定。
  4. Otsu Threshold：依 histogram 找出最佳門檻值。
  5. Binary Mask：把 pixel 分成 foreground 與 background。
  6. Auto-Polarity Correction：判斷是否需要反相，避免前景背景顛倒。
  7. 8-Connected CCL：找出每個 connected component。
  8. Bounding Box：對每個 component 計算 Xmin、Xmax、Ymin、Ymax。
  9. Border Object Rejection：接觸影像邊界的物件不畫框。
  10. Overlay：將 bounding box 邊界畫成綠色，寫入 AnsSRAM。

七、報告可用說明文字

本設計以 drawBbox 作為主要 DUT，透過固定 I/O 介面與 testbench 中的 ImgROM、UrSRAM 以及 AnsSRAM 連接。ImgROM 提供原始 RGB 影像資料，drawBbox 依序完成灰階轉換、濾波、threshold、binary mask、connected component labeling、bounding box 計算與 overlay 輸出。最終結果會寫入 AnsSRAM，由 testbench 進行結果比對。設計內部可依需求切分為多個子模組，例如 ccl_engine，但最外層 port 必須完全符合規格，以確保能與官方測試平台正確連接。

八、提交前檢查重點

  • 確認 drawBbox.v 的 module header 與作業規格完全一致。
  • 若使用 ccl_engine 或其他子模組，所有 .v/.sv 都必須放入 tar 檔。
  • 確認 Img_CEN、Ur_CEN、Ans_CEN 的 active-low 行為正確。
  • 確認 Ans_WEN=0 時才寫入最終結果。
  • 確認 done 只在整張圖與所有輸出都完成後才拉高。
