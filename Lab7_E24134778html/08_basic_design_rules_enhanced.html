08. 八、Basic Design Rules

本頁將原本的基本規則整理成可直接放進報告的詳細版本，補上同步設計原則、Reset/Enable/Done 行為、ROM/SRAM 控制規則、常見錯誤與驗證檢查表，並加入視覺化時序與 I/O 資料流圖。

一、題目說明與設計重點

本題要求說明 Lab7 硬體實作必須遵守的基本設計規則。由於 drawBbox 是以 clock-driven 的 RTL 電路完成影像讀取、灰階處理、Gaussian filter、threshold、CCL、bounding box 與 overlay，因此所有流程控制都必須依照同步設計原則，在 posedge clk 進行狀態轉移與暫存器更新。Reset 時需讓 FSM、counter、address、memory control signal 與 output data 回到安全初始值；Enable 後才可開始影像處理；全部 pixel 寫入完成後以 done=1 通知 testbench。

1

同步更新

所有暫存器、FSM 狀態、pixel counter、row/col index 與中間結果都應在 posedge clk 更新，避免 race condition 與不可預期的 latch。

2

安全 Reset

rst 為 high 時，必須清除內部流程，並將 memory control 設成不誤寫的安全狀態，例如 CEN=1、WEN=1。

3

Enable 啟動

只有 enable=1 後才能從 idle 進入 processing，避免 testbench 尚未準備好時就讀取 ROM 或寫入 SRAM。

4

Done 結束

所有 256×256 pixels 完成輸出並寫入 AnsSRAM 後，才可將 done 拉高，代表整個 pattern 驗證完成。

三、基本設計規則詳細說明

規則設計要求原因與說明
Clock Rule所有 sequential logic 使用 always @(posedge clk or posedge rst) 或等價同步流程。確保 FSM 與資料路徑在同一時脈邊緣更新，避免模擬與合成結果不一致。
Reset Rulerst=1 時清除狀態、counter、address、done 與暫存資料。若 reset 後訊號不明確，AnsSRAM 可能被誤寫，或 FSM 從未知狀態開始。
Enable Ruleenable=1 才能開始處理；enable=0 時保持 idle。配合 testbench 啟動流程，確保讀取與寫入時間點正確。
I/O Port Rule不可修改 drawBbox module 的 I/O port 名稱、位寬與方向。testbench 已固定連接 top module，port 不一致會造成 compile error 或連接錯誤。
Memory RuleROM/SRAM 的 CEN 皆為 active low；SRAM WEN=0 為寫入、WEN=1 為讀取。這是本 Lab 記憶體介面的核心規則，若極性弄反會造成輸出全錯。
Done Rule最後一個有效 pixel 已寫入後才可拉高 done。太早 done 會使 testbench 在輸出尚未完成時開始比對，導致 mismatch。

四、控制訊號與 Active 狀態

Signal方向/位元Active 狀態詳細功能設計注意事項
clkinput / 1posedge所有暫存器與 FSM 狀態更新的時脈來源。不可使用 negedge 或混合多時脈設計。
rstinput / 1High初始化整個 drawBbox 系統。Reset 時 done 應為 0，memory 寫入需關閉。
enableinput / 1High通知模組可以開始處理影像。通常由 IDLE state 判斷，enable 後進入第一個讀取狀態。
Img_CENoutput / 1LowImgROM read enable。讀取 input RGB pixel 時設為 0；不讀取時設為 1。
Img_Aoutput / 16Address指定 ImgROM 的 pixel address。256×256 共 65536 pixels，可用 addr = row*256 + col。
Img_Qinput / 32Data從 ImgROM 讀回 32-bit RGB pixel。常見格式為 00_RR_GG_BB，需依作業規格切出 R/G/B。
Ans_CENoutput / 1LowAnsSRAM chip enable。讀或寫 AnsSRAM 時皆需為 0。
Ans_WENoutput / 10 write / 1 read控制 AnsSRAM 讀寫方向。寫輸出影像時必須 Ans_CEN=0 且 Ans_WEN=0。
Ans_Aoutput / 16Address指定 AnsSRAM 寫入或讀取位置。應與輸出 pixel 的 row-major 順序一致。
Ans_Doutput / 32Data寫入 AnsSRAM 的 32-bit pixel。Bounding box 邊界 pixel 應寫入綠色 32'h0000_FF00。
doneoutput / 1High表示整張影像處理完成。done 拉高後不應再改寫 AnsSRAM。
重點記憶：ROM 只有讀取，所以看 Img_CEN；SRAM 有讀寫，所以同時看 CEN 與 WEN。 寫入 AnsSRAM 時一定是 CEN=0 且 WEN=0。

六、建議 Reset 與預設輸出寫法

以下為報告可用的 RTL 觀念範例。重點不是要求完全照抄，而是說明 reset 與 idle 狀態下應避免 SRAM 被誤寫。

// Reset / default safe control concept
if (rst) begin
    state   <= S_IDLE;
    done    <= 1'b0;
    Img_CEN <= 1'b1;   // 不讀 ROM
    Ans_CEN <= 1'b1;   // 不啟用 SRAM
    Ans_WEN <= 1'b1;   // 預設 read，避免誤寫
    Ur_CEN  <= 1'b1;
    Ur_WEN  <= 1'b1;
    Img_A   <= 16'd0;
    Ans_A   <= 16'd0;
    Ans_D   <= 32'd0;
end

七、常見錯誤與避免方式

常見錯誤可能結果修正方式
把 CEN 當成 active highROM/SRAM 沒有被正確啟用，輸出可能全黑或全錯。記住 CEN=0 才啟用。
把 WEN=1 當寫入AnsSRAM 沒有寫入結果，testbench 讀到舊資料。寫入一定是 WEN=0。
done 太早拉高尚未完成所有 pixel，造成 golden mismatch。最後一筆有效寫入完成後再 done。
Reset 未清 address/counter下一個 pattern 可能從錯誤位置開始。Reset 時所有 row、col、addr、label index 都歸零。
Combinational block 未給完整 default合成出 latch，Superlint 或 gate-level simulation 可能失敗。在 combinational always block 先給 default value。

九、報告撰寫版總結

本設計遵守同步式 RTL 設計規則，所有 FSM 狀態轉移、counter 遞增、address 更新與輸出資料寫入皆由 posedge clk 控制。Reset 階段會將所有控制訊號設為安全狀態，避免誤讀或誤寫；Enable 後系統才開始依序讀取 ImgROM、執行影像處理 pipeline，並將結果寫入 AnsSRAM。由於 ImgROM 與 SRAM 皆使用 active-low CEN，且 SRAM 寫入需使用 WEN=0，因此記憶體控制訊號的極性是本題最重要的設計規則。最後，當全部輸出 pixel 已完成寫入後，系統才拉高 done，使 testbench 能正確比對 golden result。

十、提交前檢查清單

  • ☑ drawBbox I/O port 名稱、方向與位寬沒有更改。
  • ☑ rst 可正確清除 FSM、counter、address、done 與 memory controls。
  • ☑ enable 後才開始讀取與處理影像。
  • ☑ ImgROM 讀取時 Img_CEN=0，且 address 使用 row-major mapping。
  • ☑ AnsSRAM 寫入時 Ans_CEN=0、Ans_WEN=0。
  • ☑ 不寫 SRAM 時保持安全狀態，避免多寫一筆或覆蓋結果。
  • ☑ 最後一個 pixel 寫入完成後才將 done 拉高。
  • ☑ P1～P4 simulation 皆為 PASS，且 total errors 為 0。
