03. 三、Deliverables 與檔案階層

本頁整理 Lab7 最終繳交內容、不可上傳項目、建議檔案階層，以及助教驗收時最容易檢查的重點。目標是讓提交包可以在助教環境中直接重新編譯、模擬、合成與比對結果。

一、題目說明與核心要求

本題要求說明作業繳交內容與檔案規範。Lab7 的繳交重點不是只交 drawBbox.v，而是要讓助教能用同一份提交包完成 RTL simulation、synthesis、post-synthesis simulation、結果圖片檢查與報告審查。因此，所有必要 Verilog codes、testbench、合成相關檔案、BMP 結果圖與 log 都要整理清楚。

1

可編譯

所有 RTL 檔案與子模組需完整放入，避免助教執行 vlog 時找不到 module。

2

可模擬

需包含 tb.sv、CYCLE_MAX.sv 與 pattern 對應資料，使 P1~P4 可重新驗證。

3

可合成

若作業要求 gate-level simulation，需保留合成輸出 drawBbox_syn.v 與 .sdf。

!

不可亂交

報告不可貼完整 source code，也不可上傳 .fsdb 或 .vcd waveform。

三、Deliverables 詳細表

檔案 / 類別是否需要用途與說明缺少時可能問題
drawBbox.v必交主要 DUT，負責整合影像讀取、灰階、濾波、threshold、CCL、bbox 與 overlay 控制。無法編譯，作業核心缺失。
Submodules：*.v / *.sv若有必交若 top module 內 instance 其他模組，所有被呼叫的 module 都必須提交。出現 unresolved module，可能直接 0 分。
tb.sv必交Testbench，用於讀取 pattern、驅動 DUT、比對 golden result 與輸出 BMP。助教無法重現模擬流程。
CYCLE_MAX.sv必交設定最大 cycle，避免設計卡住時 simulation 無限執行。timeout 檢查不一致。
input_rgb1.txt ~ input_rgb4.txt建議附上Pattern 原始 RGB image data，供 ImgROM 載入。無法重跑 pattern。
bbox_golden1.txt ~ bbox_golden4.txt建議附上Golden output，用於逐 pixel 比對。無法驗證輸出正確率。
drawBbox_syn.v / drawBbox_syn.sdf合成後需要Gate-level simulation 使用，確認合成後功能仍正確。無法完成 post-synthesis 驗證。
drawBbox_result_p1.bmp ~ p4.bmp建議附上四組 pattern 的最終輸出影像，可視覺檢查 bounding box 是否正確。報告缺少結果圖。
terminal_result*.log建議附上ModelSim 執行紀錄，證明 P1~P4 是否 PASS、error 數量是否為 0。無法快速佐證結果。
.fsdb / .vcd不要上傳Waveform 檔通常很大，且題目明確禁止上傳。違反繳交規範。

四、最終提交前檢查清單

檔案完整性檢查

  • 確認 drawBbox.v 與所有被 instance 的 module 都在提交包中。
  • 確認 tb.sv、CYCLE_MAX.sv 可與 RTL 一起編譯。
  • 確認 P1~P4 對應 input 與 golden 檔案名稱正確。
  • 確認合成後檔案與 timing annotation 需要的 .sdf 存在。

驗證結果檢查

  • RTL simulation：P1、P2、P3、P4 全部 PASS。
  • Total errors = 0，不能只看其中一個 pattern。
  • BMP 結果圖中 bounding box 為綠色線框，不是填滿。
  • 確認沒有把 waveform 或暫存 work library 打包進最終提交。

五、容易扣分的提交錯誤

錯誤 1：只交 drawBbox.v，但程式裡面 instance 了其他 module，導致助教編譯時找不到 module。
錯誤 2：把 .fsdb、.vcd、work/ 這類大型模擬中間檔放進去。
錯誤 3：報告中貼滿完整 Verilog source code。報告應呈現設計概念、模組架構、FSM、驗證結果與效能分析。

七、報告可直接貼上的完整文字

本作業提交檔案需包含可重新編譯與驗證的完整 RTL 設計、testbench、合成檔、模擬結果圖、terminal log 與報告文件。核心設計檔為 drawBbox.v，若設計內部使用任何子模組，也必須將所有子模組一併放入提交包中，以避免助教環境發生 compilation error。測試相關檔案包含 tb.sv、CYCLE_MAX.sv、輸入影像資料與 golden result，目的在於讓 P1~P4 pattern 可以被完整重現並進行 pixel-level comparison。合成後若需進行 gate-level simulation，則應附上 drawBbox_syn.v 與 drawBbox_syn.sdf。此外，報告中應呈現系統架構、FSM、memory control、simulation result 與 bounding box output，而不應放入完整 source code。最終提交時亦不可上傳 waveform 檔案，例如 .fsdb 或 .vcd，以符合 Lab7 deliverables 規定。

八、簡短結論

Deliverables 的重點是「完整、可重現、可驗證」。只要提交包能在乾淨環境中完成 compile、simulation、synthesis 與 result comparison，並且報告清楚呈現設計方法與結果，就能降低因缺檔或格式錯誤造成的扣分風險。
