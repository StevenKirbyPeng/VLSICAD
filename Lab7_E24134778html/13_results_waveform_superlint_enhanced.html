三、Waveform 說明：Memory Read / Write Timing

本實驗與一般純 combinational logic 最大的差異在於：影像資料必須透過 ROM/SRAM 逐 pixel 讀寫。因此 waveform 檢查重點不是只有 done 是否拉高，而是要確認 CEN、WEN、ADDR 與資料端 D/Q 是否在正確 clock cycle 對齊。

RAM / SRAM read-write waveform concept：CEN active low，WEN=1 read，WEN=0 write。
訊號Read OperationWrite Operation檢查重點
CEN0 啟用 memory0 啟用 memory若 CEN 沒有拉低，該 cycle 不會讀寫有效資料。
WEN1 表示讀取0 表示寫入WEN 極性若寫反，會造成 AnsSRAM 全黑、全白或輸出不更新。
ADDR指定讀取 pixel 位置指定寫入 pixel 位置address 必須對應 row-major：addr = y × 256 + x。
D/QQ 端輸出資料D 端輸入資料資料通常需配合 memory latency 延後一個 cycle 使用。
常見錯誤：若輸出圖片整體偏移一個 pixel 或一列，通常不是 bounding box 演算法錯，而是 ROM/SRAM read latency 或 address 更新時機錯一拍。

四、SuperLint Coverage 與可合成性檢查

SuperLint 的目的不是驗證影像演算法本身，而是檢查 RTL coding style、未使用訊號、可能的 latch、reset 行為、case completeness、位元寬度不一致等問題。作業要求所有 Verilog code 至少需達到 90% SuperLint Coverage，因此本段可用來說明程式已通過基本硬體設計規範檢查。

六、Optimization：硬體最佳化與設計理由

本設計不是單純把軟體演算法直接搬到硬體，而是必須考慮 clock cycle、memory access、combinational path、register usage 與可合成性。以下是本設計中較重要的最佳化方向：

最佳化項目做法原因與效果
FSM stage partition將 Load、Gray、Gaussian、Otsu、Mask、CCL、BBox、Draw 分成不同 state。避免單一 state 內組合邏輯過長，降低 timing violation 風險。
Shift instead of divisionGaussian kernel 權重總和為 16，可使用 >> 4 取代除法。硬體除法器成本高，shift 可降低面積與延遲。
Memory reuse保留原始 RGB 資料，最後 overlay 時只改 bbox 邊界 pixel。避免重複讀取 ImgROM，並保留原圖視覺品質。
Union-Find equivalence tableCCL 中使用 parent table 管理等價 label。可處理多個 label 後續合併的情況，提升連通元件判斷正確性。
Boundary object filtering接觸影像邊界的物件不畫框。避免背景或破碎邊緣被誤判成完整物件。
Overlay overlap handling多個 bbox 邊界重疊時，該 pixel 仍維持綠色輸出。確保重疊區域不會被原圖覆蓋，也不會造成多重寫入錯誤。

七、Lessons Learned：實驗心得

透過本次 Lab7，我更清楚理解影像處理演算法與硬體實作之間的差異。在軟體中，RGB 轉灰階、Gaussian filter、threshold 或 CCL 可以用陣列與迴圈快速完成；但在硬體設計中，每一次 memory read、每一次 address 更新、每一個 state transition 都會受到 clock cycle 影響。因此，只要 ROM/SRAM 的 CEN、WEN 或 address 時序差一拍，最後輸出的 BMP 就可能產生整張偏移、pixel mismatch，甚至 bounding box 畫在錯誤位置。

此外，本實驗也讓我學到完整 VLSI CAD flow 的重要性。RTL simulation 可以先確認功能正確，SuperLint 可以檢查 coding style 與潛在硬體問題，synthesis 則可以進一步觀察電路面積、timing 與 gate-level behavior。尤其 Draw Bounding Box 並不是單一演算法，而是由多個階段串接而成，因此最有效的 debug 方法是分階段檢查：先確認輸入影像讀取正確，再確認灰階與濾波結果，接著檢查 mask、CCL label、bbox 座標，最後才檢查 overlay 輸出。

整體而言，本次作業讓我熟悉如何把高階影像處理觀念轉換為可合成的 Verilog FSM 架構，也學會如何用 terminal result、waveform、BMP 圖片與 SuperLint report 來證明設計的正確性與完整性。
