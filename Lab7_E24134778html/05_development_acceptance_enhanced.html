一、題目說明與開發目標

Lab7 的核心目標是完成一個可合成的影像處理硬體模組 drawBbox。系統需讀取 256×256 RGB 影像，依序完成灰階轉換、Gaussian 平滑、Otsu threshold、binary mask、auto-polarity correction、8-connected CCL、Bounding Box 計算與綠色框線 overlay，最後將結果寫入 AnsSRAM。

256×256
固定影像大小

所有 address、row、col 與 SRAM 寫入位置都必須對應到 65536 個 pixels。

32-bit
RGB / SRAM 資料寬度

輸入與輸出皆以 32-bit unsigned data 表示，需避免 port width mismatch。

8-Conn
連通元件標記

CCL pass1 需考慮 NW、N、NE、W，使斜向接觸仍可被視為同一物件。

0 Errors
滿分驗證目標

P1~P4 與 golden output 比對時，每個 pixel 都需完全一致。

二、開發流程總覽圖

整體開發建議採用「先單元、再整合、最後驗證」策略。每個階段完成後都先用小型測試或輸出檔案確認資料正確，再接到下一個 stage，避免最後才發現錯誤難以追蹤。

三、逐階段開發需求與驗證重點

步驟開發內容實作要求驗證重點
1ImgROM 讀圖依 row-major 掃描整張影像，位址公式為 addr = row × 256 + col。確認 Img_CEN active-low，讀取資料延遲與 FSM 對齊。
2RGB to Gray將 32-bit RGB pixel 轉為單一灰階值，並處理 rounding。灰階公式、位元切割、四捨五入與 overflow 保護。
3Gaussian Filter使用 3×3 convolution 平滑雜訊，提升 threshold 穩定性。kernel 權重、邊界 padding、除法/shift 是否正確。
4Otsu Threshold建立 256-level histogram，找出最大類間變異數 threshold。累加器位寬、除以 0 保護、threshold 更新與 final T。
5Binary Mask / Auto Polarity以 threshold 產生 0/1 mask，必要時翻轉前景背景。前景比例是否合理，避免 foreground/background 顛倒。
68-connected CCL檢查 NW、N、NE、W 已掃描鄰居，使用 label equivalence 合併。斜向連通、union-find root、label merge 後是否同群。
7Bounding Box對每個 component 記錄 Xmin、Xmax、Ymin、Ymax。座標是否 row/col 對調、邊界物件是否需過濾。
8Overlay / Draw保留原始 RGB，只在框線 pixel 寫入綠色 32'h0000_FF00。只畫 1-pixel 邊界，不可填滿框內區域。
9Compare / Regression將輸出與 bbox_golden1~4.txt 比對。P1~P4 全部 errors = 0，才代表 pixel-perfect。
開發建議：不要一次把所有功能寫完才測試。最穩定的方法是先確認 memory control，再驗證灰階與濾波，再做 threshold / mask，最後接 CCL 與 overlay。這樣 debug 時可以快速定位問題。

六、常見錯誤與避免方式

1. 影像偏移一格
通常來自 ROM read latency 或 row/col 更新時機不一致。解法是建立 read-wait / capture state，讓 address 與 Img_Q 對齊。
2. 綠框畫成填滿矩形
Bounding Box overlay 只應在 x==xmin/xmax 或 y==ymin/ymax 的 pixel 寫綠色，框內其他 pixel 必須保留原始 RGB。
3. CCL label 沒有正確合併
8-connected CCL 需要檢查 NW、N、NE、W。若只檢查 N、W，斜向連接物件會被拆成多個 component。
4. 邊界物件誤畫框
若 component 碰到 image border，依規則可能不需繪製 bounding box，否則 golden comparison 會出現大量 mismatch。
5. 合成失敗或 lint 分數低
避免不可合成語法、未指定 default assignment、過大的 combinational block，以及未使用或重複宣告的 signal。

  • □ 輸出檢查：drawBbox_result_p1.bmp 至 drawBbox_result_p4.bmp 均可正常開啟。
  • □ 檔案檢查：所有 Verilog source、testbench、input txt、golden txt、Makefile、synthesis script 均放入提交包。
  • □ 合成檢查：Design Compiler 可產生 netlist 與 SDF，gate-level simulation 無功能錯誤。
  • □ Lint 檢查：SuperLint coverage 達 90% 以上，主要 warning 已修正或能合理解釋。
  • □ 報告檢查：報告只放設計說明、圖、結果與分析，不直接貼完整 source code。
