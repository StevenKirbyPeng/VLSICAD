04. 四、作業說明與目的

本頁說明 Lab7 的核心任務：以 Verilog 設計一個可合成的硬體影像處理引擎，從 256×256 RGB 影像中分離前景物件、完成 8-connected CCL，並將正確的綠色 Bounding Box 疊回輸出影像。

一、題目說明

本題要求解釋 Lab7 的作業目標與設計價值。Lab7 不是單純畫圖，而是把「影像處理演算法」轉換成「可在硬體中逐 clock 執行的 RTL 流程」。整個系統必須從 ImgROM 讀取原始 32-bit RGB pixel，經過灰階化、Gaussian 濾波、Otsu 門檻、Mask 產生、8-connected CCL、Bounding Box 計算，最後把綠色邊界框寫入 AnsSRAM。

二、完整答案：Lab7 的作業目的

本作業的主要目標，是完成一個能在硬體中自動處理影像的 Bounding Box Engine。輸入端提供 256×256 的 RGB 影像，每個 pixel 以 32-bit 格式儲存在 ImgROM 中。系統必須依序讀取每一個 pixel，將彩色資訊轉為灰階亮度，再透過 3×3 Gaussian filter 平滑化影像，使雜訊不會在後續 threshold 與 CCL 階段造成誤判。接著，系統使用 Otsu threshold 從灰階統計分布中自動找出適合的門檻值，將影像轉成 binary mask。產生 mask 後，再透過 8-connected Connected Component Labeling 將相鄰的前景 pixel 合併為同一個物件，並對每個物件計算最小與最大座標，形成 Bounding Box。

最後，設計需將計算出的邊界框以綠色 32'h0000_FF00 疊加回原始影像，只修改邊界線上的 pixel，其餘位置保留原始 RGB 顏色。完成整張影像後，系統將輸出寫入 AnsSRAM，並拉高 done 訊號通知 testbench 檢查結果。此作業的重點不只是功能正確，還包含 synchronous design、記憶體時序控制、pixel-level accuracy、RTL simulation、post-synthesis simulation 與 SuperLint coverage 等完整 VLSI CAD 設計流程。

1影像前處理

2自動門檻分割

38-connected CCL
支援斜向連通，避免同一個物體因邊緣破碎而被切成多個 component。
4Bounding Box Overlay
只在物件外框畫 1-pixel 綠色線條，保留原圖內容並輸出到 AnsSRAM。
