一、題目說明

本題 Understanding the Function 要求說明系統在初始化後，如何從輸入影像一路完成灰階化、濾波、二值化、連通元件標記、Bounding Box 計算與 Overlay 輸出。重點不是只列出模組名稱，而是要說清楚每一步的輸入、處理目的、輸出資料，以及它在硬體流程中扮演的角色。

報告撰寫重點：Lab7 的核心是將 256×256 RGB 影像轉成可辨識物件位置的結果。最後輸出不是純 mask，而是保留原圖，並在偵測到的物件外框畫上綠色 bounding box。

系統可視為一條由影像前處理、物件分割、幾何分析與結果輸出組成的硬體 pipeline。輸入資料由 ImgROM 提供，處理後結果寫入 AnsSRAM。整體流程如下：

三、完整解答：逐步功能詳解

Step 1 — Read and Convert

系統啟動後，先從 testbench 的 ImgROM 讀取原始 RGB pixels。影像大小為 256×256，總共有 65536 pixels。位址採用 row-major mapping：

addr = row × 256 + col

每個 pixel 為 32-bit 格式，主要使用其中的 R、G、B 三個 8-bit channel。讀入後，系統將 RGB 轉成灰階值，目的是把三通道資料壓縮成單一亮度資訊，使 threshold 與 histogram 統計可以用一個 intensity value 完成。

Gray ≈ 0.299R + 0.587G + 0.114B

Step 2 — Filter Noise

灰階影像經過 3×3 Gaussian blur filter。此步驟會把中心 pixel 與周圍 8 個鄰居做加權平均，讓局部亮度變化更平滑。對硬體影像處理來說，這能減少雜訊造成的孤立前景點，避免 CCL 後產生太多假物件。

Kernel = (1/16) × [[1,2,1],[2,4,2],[1,2,1]]

在邊界位置，因為 3×3 window 會超出影像範圍，因此需搭配邊界處理策略，例如 Reflect101 padding，讓最外圈 pixel 也能得到合理的濾波結果。

Step 3 — Binarization by Otsu Threshold

Gaussian-filtered grayscale image 會被拿來建立 0~255 的 histogram。Otsu 方法會掃描所有可能 threshold，尋找能讓前景與背景分離最明顯的門檻。其核心概念是最大化類間變異數：

σB2 = WbWf(μb − μf)2

找到 threshold T 後，系統使用下列規則產生 binary mask：

mask(x,y) = 1 if gray(x,y) > T, else 0

Step 4 — Auto-Polarity Correction

不同測試圖片中，物件可能比背景亮，也可能比背景暗。若直接用 gray > T 判斷，有些 pattern 會出現前景與背景顛倒的情況。因此系統會統計高於 threshold 的 pixel 數量，若比例不合理，就進行 polarity correction：

mask'(x,y) = 1 − mask(x,y)

這個步驟確保後續 CCL 所看到的 1 代表真正需要分析的 foreground object。

Step 5 — CCL and Bounding Box Extraction

系統使用 8-connected CCL 將相連之前景 pixels 分成不同 connected components。掃描時只需要看已經掃描過的鄰居，也就是 NW、N、NE、W。若鄰居 label 不同，則透過 union-find 建立等價關係。

Neighbor set = {NW, N, NE, W}

完成 label 分群後，系統對每個 component 計算最小與最大座標：

Xmin, Xmax, Ymin, Ymax

這四個值決定了 Bounding Box 的左、右、上、下邊界。

Step 6 — Overlay and Draw

最後階段不是輸出二值圖，而是把 bounding box 疊回原本的 RGB 影像。若目前 pixel 位於任一有效 bounding box 的邊界上，則寫入綠色；否則保留原始 RGB pixel。

GREEN = 32'h0000_FF00

若多個 bounding box 的邊界剛好落在同一 pixel，該 pixel 仍只需寫入一次綠色。這種 overlap 不需要重複計算顏色，因為最終輸出只檢查該 pixel 是否為 bounding box 邊界。

Step 7 — Finish

當所有 pixel 都已寫入 AnsSRAM 後，系統將 done 拉高，通知 testbench 可以開始比對輸出影像。若每個 pixel 都與 golden result 一致，該 pattern 即通過。

七、可直接放入報告的精簡版答案

After initialization, the system reads the 256×256 RGB image from ImgROM in row-major order and converts each 32-bit RGB pixel into grayscale intensity. The grayscale image is then processed by a 3×3 Gaussian filter to reduce noise and stabilize the following segmentation stage. Next, an Otsu threshold is calculated from the gray-level histogram, and a binary mask is generated. If the foreground and background polarity is reversed, the mask is automatically corrected. The system then performs 8-connected component labeling by checking NW, N, NE, and W neighbors and groups connected foreground pixels into components. For each valid component, the minimum and maximum x/y coordinates are recorded to form a bounding box. Finally, the original RGB image is written to AnsSRAM, while the pixels located on bounding box boundaries are replaced with green color 32'h0000_FF00. When all pixels are written, done is asserted to indicate completion.
